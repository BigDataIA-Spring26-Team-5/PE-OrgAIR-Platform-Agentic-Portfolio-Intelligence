  ---
  How It Works — Summary

  Phase 1 — Signal Collection (CS2)

  Six parallel background tasks hit external APIs and produce normalized [0-100] scores with confidence values:

  ┌─────────────────────┬───────────────────────────────┬───────────────────────────────────────────┐
  │       Signal        │            Source             │             What it measures              │
  ├─────────────────────┼───────────────────────────────┼───────────────────────────────────────────┤
  │ technology_hiring   │ JobSpy (LinkedIn/Indeed)      │ AI job ratio within tech postings         │
  ├─────────────────────┼───────────────────────────────┼───────────────────────────────────────────┤
  │ innovation_activity │ PatentsView / USPTO           │ AI patent ratio + recency                 │
  ├─────────────────────┼───────────────────────────────┼───────────────────────────────────────────┤
  │ digital_presence    │ BuiltWith + Wappalyzer + Groq │ AI tools in tech stack                    │
  ├─────────────────────┼───────────────────────────────┼───────────────────────────────────────────┤
  │ leadership_signals  │ SEC DEF-14A proxy filings     │ Tech exec presence + AI strategy keywords │
  ├─────────────────────┼───────────────────────────────┼───────────────────────────────────────────┤
  │ board_composition   │ SEC proxy (board bios)        │ Board-level tech expertise                │
  ├─────────────────────┼───────────────────────────────┼───────────────────────────────────────────┤
  │ culture             │ Glassdoor reviews (S3)        │ Culture indicators for AI readiness       │
  └─────────────────────┴───────────────────────────────┴───────────────────────────────────────────┘

  Each signal uses a weighted sub-scoring formula (ratio + volume + diversity components) then persists to Snowflake.

  ---
  Phase 2 — Evidence-to-Dimension Mapping (CS3 early)

  The Evidence Mapper (evidence_mapper.py) implements the Table 1 Matrix: each of the 9 evidence sources (6 signals + 3 SEC section rubric scores) distributes its score
  across the 7 V^R dimensions using primary/secondary weights. For example, technology_hiring contributes 70% to talent_skills, 20% to technology_stack, 10% to
  data_infrastructure.

  The Rubric Scorer (rubric_scorer.py) processes raw SEC text (Items 1, 1A, 7) through 5-level keyword-matching rubrics to produce 3 additional input scores.

  ---
  Phase 3 — 7 Dimension Scores

  After mapping, each company has 7 weighted dimension scores:

  data_infrastructure (0.18) → talent_skills (0.17) → ai_governance (0.15) → technology_stack (0.15) → leadership_vision (0.13) → use_case_portfolio (0.12) →
  culture_change (0.10)

  ---
  Phase 4 — Composite Scoring (6-step cascade)

  1. TC — Talent Concentration: how distributed vs. siloed AI talent is (from job seniority mix + skill spread)
  2. V^R — Venture Readiness: weighted average of 7 dimensions, penalized if TC is too concentrated or scores are too unbalanced (CV penalty)
  3. PF — Position Factor: market cap percentile mapped to [-1, +1] competitive advantage
  4. H^R — Human Readiness: sector baseline (tech=84, finance=68…) adjusted by PF
  5. Synergy — how well V^R and H^R are aligned, amplified by sector timing factor
  6. Org-AI-R — Final score: 0.88 × [0.60×V^R + 0.40×H^R] + 0.12×Synergy

  The constants α=0.60, β=0.12 control how much internal readiness (V^R) vs. market position (H^R) vs. alignment synergy drives the final investment-grade score.

  │  score_improvement = target_score - entry_score                              │
  │                                                                              │
  │  Sector multipliers (bps per Org-AI-R point):                                │
  │    Tech=0.45  FinServ=0.38  Healthcare=0.35  Mfg=0.30  Retail=0.28           │
  │                                                                              │
  │  Implementation cost (% revenue per point):                                  │
  │    Tech=0.08  FinServ=0.12  Healthcare=0.15  etc.                            │
  │                                                                              │
  │  ebitda_impact = score_improvement x sector_multiplier                       │
  │  impl_cost    = score_improvement x cost_factor                              │
  │  net_impact   = ebitda_impact - impl_cost                                    │
  │                                                                              │
  │  H^R Risk Adjustment:                                                        │
  │    hr_risk_factor = min(1.0, max(0.5, h_r_score / 80))                       │
  │    H^R > 70 -> low risk (minimal adjustment)                                 │
  │    H^R < 50 -> high risk (50% floor)                                         │
  │                                                                              │
  │  adjusted_net = net_impact x hr_risk_factor                                  │
  │                                                                              │
  │  Time-to-Value: <=10pt=12mo, <=20pt=18mo, <=30pt=24mo, >30pt=36mo            │
  │  Confidence: <=15pt + H^R>=60 -> high, <=25pt -> medium, >25pt -> low        │
  │                                                                              │
  │  Output: EBITDAProjection                                                    │
  │    ├─ scenarios {conservative, base, optimistic}                             │
  │    ├─ time_to_value_months                                                   │
  │    └─ confidence {high, medium, low}                                         │
  └──────────────────────────────────────────────────────────────────────────────┘
